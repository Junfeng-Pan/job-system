# 标准岗位画像构建指南

本指南旨在说明如何从海量（约 10,000 条）原始岗位描述中，通过聚类聚合与大模型分析，构建出代表行业标准的“岗位画像”。

## 1. 核心字段定义

大模型输出的 JSON 结构需严格遵循以下定义，确保数据在后续系统中的一致性。

| 字段名 | 子字段 | 类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- | :--- |
| **summary** | - | string | **岗位综述**：对该岗位的职责核心、行业地位及主要工作内容的宏观总结。 | 该岗位是软件开发的核心力量，负责... |
| **skills** | `name` | string | **专业技能要求**：该岗位必须掌握的硬技能、工具、框架或行业知识。 | Java开发、市场数据分析 |
| | `evidence` | string | **要求摘要**：从原始数据中提取的技能应用场景或详细要求描述。 | 精通SpringBoot框架，具备微服务架构设计能力。 |
| **thresholds** | `name` | string | **基础门槛要求**：入职该岗位的刚性过滤条件（学历、经验、证书等）。 | 学历、工作年限 |
| | `evidence` | string | **要求摘要**：具体的数值或等级要求。 | 统招本科及以上学历，英语六级（CET-6）。 |
| **professionalism** | `name` | string | **职业素养要求**：软技能、工作态度、价值观及沟通协调能力。 | 抗压能力、团队协作 |
| | `evidence` | string | **要求摘要**：该素质在岗位工作中的体现形式。 | 能够适应高强度项目节奏，具备跨部门沟通推动力。 |

## 2. 聚合推理提示词 (Prompt)

### 系统提示词 (System Prompt)
```text

你的任务是根据提供的多个同类岗位的招聘描述（JD），归纳总结出该岗位的“标准画像”。
你需要过滤掉个别公司的特殊要求，提取出该职业在行业中的通用核心能力。

请严格按照以下 JSON 格式输出，不要包含任何额外的解释文本：

{
  "summary": "对该岗位的宏观定义与核心职责总结",
  "skills": [
    {
      "name": "技能名称",
      "evidence": "技能的详细要求与应用场景总结"
    }
  ],
  "thresholds": [

      "name": "门槛类别",
      "evidence": "具体的硬性指标要求"
    }
  ],
  "professionalism": [
    {
      "name": "素养名称",
      "evidence": "在该岗位中的具体体现"
    }
  ]
}
```

### 用户提示词 (User Prompt)
```text
岗位类别：{{job_category_name}}
原始描述集合：
---
{{job_description_list}}
---

请根据上述数据集合，归纳生成该岗位的行业标准画像。请确保内容专业、详实，并严格遵守 JSON 格式。
```

## 3. 构建流程建议

1.  **数据统计**：对 `jobs` 表中的 `title` 或 `industry` 进行频率统计，筛选出 Top 10 的热门岗位类别。
2.  **内容清洗**：将同一类别的 `job_detail` 进行合并。如果文本过长，建议分段处理或由 LLM 先进行初步摘要。
3.  **模型调用**：使用上述提示词调用 `qwen-plus` 级别以上的模型进行综合分析，获得结构化结果。
4.  **数据持久化**：将生成的 `summary` 以及各项特征存储在专门的 `standard_profiles` 表中，以便后续匹配算法使用。
